本指南将帮助你利用 LLM 来构建自定义的模型上下文协议（MCP）服务端和客户端。在本教程中，我们将以 Claude 为例，但你也可以使用任何领先的 LLM 实现类似功能。

## 准备文档

在开始之前，请收集必要的文档，以便 Claude 理解 MCP：

1. 访问 https://modelcontextprotocol.io/llms-full.txt，并复制完整的文档内容。
2. 打开 [MCP TypeScript SDK](https://github.com/modelcontextprotocol/typescript-sdk) 或 [Python SDK 仓库](https://github.com/modelcontextprotocol/python-sdk)。
3. 复制其中的 README 文件或其他相关文档。
4. 将这些文档粘贴到你与 Claude 的对话中。

## 描述你的服务器目标

在提供相关文档后，清晰地向 Claude 描述你想要构建的服务器类型。具体描述以下内容：

- 服务器将暴露的资源
- 它将提供的工具
- 它应包含的提示
- 需要交互的外部系统

例如：
```
构建一个 MCP 服务器，要求如下：
- 连接到我公司内部的 PostgreSQL 数据库
- 将数据表结构作为资源暴露出来
- 提供运行只读 SQL 查询的工具
- 包含常见数据分析任务的提示
```

## 与 Claude 一起工作

当和 Claude 一起开发 MCP 服务器时：

1. 先实现核心功能，然后逐步迭代添加更多特性。
2. 请 Claude 解释代码中你不理解的部分。
3. 根据需要请求修改或改进。
4. 让 Claude 帮助你测试服务器并处理特殊情况。

Claude 能帮助你实现以下关键 MCP 功能：

- 资源管理和暴露
- 工具定义和实现
- 提示模板和处理器
- 错误处理和日志记录
- 连接和传输设置

## 最佳实践

使用 Claude 构建 MCP 服务器时，请遵循以下建议：

- 将复杂的服务器分解为更小的组件。
- 在继续开发之前，对每个组件进行彻底测试。
- 考虑安全性——验证用户输入并适当限制访问。
- 为代码做好文档记录，便于未来的维护。
- 仔细遵循 MCP 协议规范。

## 下一步

在 Claude 帮助你构建服务器后：

1. 仔细审核生成的代码。
2. 使用 MCP Inspector 工具测试你的服务器。
3. 将服务器连接到 Claude.app 或其他 MCP 客户端。
4. 根据实际使用反馈进行迭代改进。

请记住，Claude 可以根据需求的变化，继续帮你修改和改进你的服务器。

需要更多帮助？随时向 Claude 提问，获取关于实现 MCP 功能或解决问题的具体指导。
