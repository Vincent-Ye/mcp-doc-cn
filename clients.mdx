## 客户端详情

### Claude 桌面应用
Claude 桌面应用程序提供对 MCP 的全面支持，实现与本地工具和数据源的深度集成。

**关键功能：**  
- 完整支持资源功能，可以附加本地文件和数据  
- 支持提示模板  
- 工具集成，用于执行命令和脚本  
- 本地服务器连接，增强隐私和安全性  

> ⓘ 注意：Claude.ai 的网页应用当前不支持 MCP。MCP 功能仅在桌面应用程序中可用。

### 5ire
[5ire](https://github.com/nanbingxyz/5ire) 是一个开源的跨平台桌面 AI 助手，通过 MCP 服务器支持各种工具。

**关键功能：**  
- 内置 MCP 服务器，可快速启用和禁用。  
- 用户可以通过修改配置文件添加更多服务器。  
- 开源且用户友好，适合初学者使用。  
- 未来将持续改进对 MCP 的支持。

### BeeAI 框架
[BeeAI 框架](https://i-am-bee.github.io/beeai-framework) 是一个用于大规模构建、部署和提供强大代理工作流的开源框架。该框架包含 **MCP 工具**，一种原生功能，可简化将 MCP 服务器集成到代理工作流中的过程。

**关键功能：**  
- 将 MCP 工具无缝集成到代理工作流中。  
- 从连接的 MCP 客户端快速实例化框架原生工具。  
- 计划未来支持代理级 MCP 功能。  

**了解更多：**  
- [在代理工作流中使用 MCP 工具的示例](https://i-am-bee.github.io/beeai-framework/#/typescript/tools?id=using-the-mcptool-class)

### Cline
[Cline](https://github.com/cline/cline) 是 VS Code 中的一个自主编码代理，能够按照每一步用户许可编辑文件、运行命令、使用浏览器等。

**关键功能：**  
- 使用自然语言创建和添加工具 (例如："添加一个搜索网页的工具")  
- 通过 `~/Documents/Cline/MCP` 目录与他人分享由 Cline 创建的自定义 MCP 服务器  
- 显示已配置的 MCP 服务器及其工具、资源和错误日志  

### Continue
[Continue](https://github.com/continuedev/continue) 是一个开源的 AI 代码助手，内置对所有 MCP 功能的支持。

**关键功能：**  
- 输入 "@" 来提到 MCP 资源  
- 提示模板以斜杠命令形式呈现  
- 直接在聊天中使用内置工具和 MCP 工具  
- 支持 VS Code 和 JetBrains IDE，适配任何 LLM  

### Cursor
[Cursor](https://docs.cursor.com/advanced/model-context-protocol) 是一个 AI 代码编辑器。  

**主要功能**：
- 支持在 Cursor Composer 中使用 MCP 工具
- 同时支持 STDIO 和 SSE

### Emacs Mcp
[Emacs Mcp](https://github.com/lizqwerscott/mcp.el) 是一个为 Emacs 设计的客户端，用于与 MCP 服务器交互，实现无缝连接和操作。它为像 [gptel](https://github.com/karthink/gptel) 和 [llm](https://github.com/ahyatt/llm) 这样的 AI 插件提供了 MCP 工具调用支持，并遵循 Emacs 标准的工具调用格式。此集成增强了 Emacs 生态系统中 AI 工具的功能。

**主要功能**：
- 为 Emacs 提供 MCP 工具支持。

### Firebase Genkit
[Genkit](https://github.com/firebase/genkit) 是 Firebase 推出的 SDK，用于构建和集成生成式人工智能（GenAI）功能到应用程序中。[genkitx-mcp](https://github.com/firebase/genkit/tree/main/js/plugins/mcp) 插件让用户能够将 MCP 服务器作为客户端使用，或者通过 Genkit 工具和提示创建 MCP 服务器。

**主要功能**：
- 为工具和提示提供客户端支持（资源支持部分实现）
- 在 Genkit 的开发者界面 playground 中提供丰富的资源发现支持
- 与 Genkit 的现有工具和提示实现无缝互操作
- 兼容多种顶级提供商的 GenAI 模型

### GenAIScript
利用 [GenAIScript](https://microsoft.github.io/genaiscript/)（基于 JavaScript）以编程方式组装 LLM 提示。在 JavaScript 中调度 LLM、工具和数据。

**主要功能**：
- JavaScript 工具箱，用于处理提示
- 提供抽象层，使操作更加轻松高效
- 与 Visual Studio Code 无缝集成

### Goose
[Goose](https://github.com/block/goose) 是一个开源 AI 代理，通过自动化编码任务来提升软件开发效率。

**主要功能**：
- 通过工具将 MCP 功能暴露给 Goose。
- MCP 可通过 [扩展目录](https://block.github.io/goose/v1/extensions/)、CLI 或 UI 直接安装。
- Goose 允许通过 [构建自定义 MCP 服务器](https://block.github.io/goose/docs/tutorials/custom-extensions) 来扩展其功能。
- 包含内置工具用于开发、网页爬取、自动化、内存管理，并支持与 JetBrains 和 Google Drive 集成。

### LibreChat
[LibreChat](https://github.com/danny-avila/LibreChat) 是一个开源、可定制的 AI 聊天界面，支持多个 AI 提供商，并现已支持 MCP 集成。

**主要功能**：
- 通过 MCP 服务器扩展当前的工具生态系统，包括 [代码解释器](https://www.librechat.ai/docs/features/code_interpreter) 和图像生成工具
- 通过多个顶级提供商的 LLM，为可定制的[代理](https://www.librechat.ai/docs/features/agents)添加工具
- 开源且支持自托管，具备安全的多用户支持
- 未来路线图包括扩展 MCP 功能支持

### mcp-agent
[mcp-agent] 是一个简单的、可组合化的框架，用于基于 Model Context Protocol 构建代理。

**主要功能**：
- 自动管理 MCP 服务器连接。
- 将来自多台服务器的工具暴露给 LLM。
- 实现了 [构建高效代理](https://www.anthropic.com/research/building-effective-agents) 中定义的所有模式。
- 支持工作流程的暂停/恢复信号，例如等待人工反馈。

### oterm
[oterm] 是一个专为 Ollama 提供的终端客户端，允许用户创建聊天/代理。

**主要功能**：

- 支持与工具连接的多个完全可自定义的 Ollama 聊天会话。
- 支持 MCP 工具。

### Roo Code
[Roo Code](https://roocode.com) 通过 MCP 提供 AI 编码协作功能。

**主要功能**：
- 支持 MCP 工具和资源
- 集成到开发工作流程中
- 可扩展的 AI 功能

### Sourcegraph Cody
[Cody](https://openctx.org/docs/providers/modelcontextprotocol) 是 Sourcegraph 推出的 AI 编码助手，通过 OpenCTX 实现 MCP。

**主要特点：**  
- 支持MCP资源  
- 集成Sourcegraph的代码智能功能  
- 使用OpenCTX作为抽象层  
- 计划未来支持额外的MCP功能  

### SpinAI  
[SpinAI](https://spinai.dev) 是一个开源TypeScript框架，用于构建可观察的AI代理。该框架提供原生的MCP兼容性，使AI代理能够与MCP服务器和工具无缝集成。  

**主要特点：**  
- 内置MCP兼容性，适用于AI代理  
- 开源TypeScript框架  
- 可观察的代理架构  
- 原生支持MCP工具集成  

### Superinterface  
[Superinterface](https://superinterface.ai) 是一个AI基础设施和开发者平台，支持构建具有MCP支持的应用内AI助手，包括交互组件、客户端函数调用等功能。  

**主要特点：**  
- 可通过React组件或脚本标签嵌入，用于助手调用MCP服务器工具  
- 支持SSE传输协议  
- 可使用任何AI提供商的AI模型（例如OpenAI、Anthropic、Ollama等）  

### TheiaAI/TheiaIDE  
[Theia AI](https://eclipsesource.com/blogs/2024/10/07/introducing-theia-ai/) 是一个用于构建AI增强工具和IDE的框架。[AI驱动的Theia IDE](https://eclipsesource.com/blogs/2024/10/08/introducting-ai-theia-ide/) 是基于Theia AI构建的开放且灵活的开发环境。  

**主要特点：**  
- **工具集成**：Theia AI允许AI代理（包括Theia IDE中的代理）利用MCP服务器实现工具的无缝交互。  
- **可自定义提示词**：Theia IDE用户可定义和调整提示词，通过动态集成MCP服务器满足定制化工作流程需求。  
- **自定义代理**：Theia IDE支持创建利用MCP功能的自定义代理，使用户能够快速设计专属工作流。  

Theia AI和Theia IDE的MCP集成为用户提供了极大的灵活性，使它们成为探索和适应MCP的强大平台。  

**了解更多：**  
- [Theia IDE 和 Theia AI MCP 支持公告](https://eclipsesource.com/blogs/2024/12/19/theia-ide-and-theia-ai-support-mcp/)  
- [下载AI驱动的Theia IDE](https://theia-ide.org/)  

### Windsurf Editor  
[Windsurf Editor](https://codeium.com/windsurf) 是一个结合AI辅助和开发者工作流的智能化IDE。它引入了创新的AI Flow系统，既支持协作式AI交互，也支持独立操作，同时确保开发者掌控权。  

**主要特点：**  
- 颠覆性的AI Flow模式，用于人类与AI协作  
- 智能代码生成和理解  
- 支持多模型的丰富开发工具  

### Zed  
[Zed](https://zed.dev/docs/assistant/model-context-protocol) 是一个高性能代码编辑器，内置MCP支持，专注于提示模板和工具集成。  

**主要特点：**  
- 提示模板以斜杠命令形式呈现在编辑器中  
- 工具集成以提升代码工作流  
- 与编辑器功能和工作区上下文紧密结合  
- 不支持MCP资源  

### OpenSumi  
[OpenSumi](https://github.com/opensumi/core) 是一个框架，帮助快速构建AI原生IDE产品。  

**主要特点：**  
- 支持在OpenSumi中使用MCP工具  
- 支持内置IDE MCP服务器以及自定义MCP服务器  

### Daydreams  
[Daydreams](https://github.com/daydreamsai/daydreams) 是一个生成式代理框架，用于执行链上操作。  

**主要特点：**  
- 在配置中支持MCP服务器  
- 提供MCP客户端接口  

## 在应用中添加MCP支持  

如果您已经为应用添加了MCP支持，欢迎提交Pull Request以将其添加到此列表中。MCP集成可为用户提供强大的上下文AI功能，同时让您的应用成为不断发展的MCP生态系统的一部分。  

添加MCP支持的优势：  
- 让用户能够带入自己的上下文和工具  
- 加入一个正在发展的可互操作AI应用生态系统  
- 向用户提供灵活的集成选项  
- 支持本地优先的AI工作流  

要在您的应用程序中开始实施MCP，请查看我们的 [Python SDK文档](https://github.com/modelcontextprotocol/python-sdk) 或 [TypeScript SDK文档](https://github.com/modelcontextprotocol/typescript-sdk)。
