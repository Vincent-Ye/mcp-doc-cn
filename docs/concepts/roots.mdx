### 什么是根目录？

根目录是 MCP 中的一种概念，用于定义服务器可以运行的边界。它为客户端提供一种方法，向服务器告知相关资源及其位置。

## 根目录的定义

根目录是客户端建议服务器关注的 URI。当客户端连接到服务器时，它会声明服务器应该处理哪些根目录。虽然根目录主要用于文件系统路径，但实际上根目录可以是任何有效的 URI，包括 HTTP URL。

例如，根目录可以是：

```
file:///home/user/projects/myapp
https://api.example.com/v1
```

## 为什么要使用根目录？

根目录具有以下几个重要作用：

1. **指导性**：它们向服务器提供关于相关资源和位置的信息。
2. **清晰性**：根目录可以明确哪些资源属于您的工作空间。
3. **组织性**：使用多个根目录可以同时管理不同资源。

## 根目录的工作原理

当客户端支持根目录时，它会：

1. 在连接时声明 `roots` 功能。
2. 向服务器提供建议的根目录列表。
3. 当根目录发生变化时（若支持），通知服务器。

虽然根目录主要是提供信息，而不强制执行，但服务器应当：

1. 尊重客户端提供的根目录。
2. 使用根目录 URI 来定位和访问资源。
3. 优先处理根目录边界内的操作。

## 常见用例

根目录通常用于定义：

- 项目目录
- 存储库位置
- API 端点
- 配置位置
- 资源边界

## 最佳实践

在使用根目录时：

1. 仅建议必要的资源。
2. 使用清晰且描述性强的根目录名称。
3. 监控根目录的可访问性。
4. 优雅地处理根目录的变化。

## 示例

以下是一个典型 MCP 客户端公开根目录的方式：

```json
{
  "roots": [
    {
      "uri": "file:///home/user/projects/frontend",
      "name": "前端仓库"
    },
    {
      "uri": "https://api.example.com/v1",
      "name": "API 端点"
    }
  ]
}
```

此配置建议服务器同时关注本地的前端仓库和一个 API 端点，同时保持它们逻辑上的分离。
